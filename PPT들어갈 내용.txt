개발 일정 

1주차 -> 프로젝트 주제와 기능 및 개발 일정 논의
2주차~3주차 -> UI/UX구현, 핵심 기능 테스트 및 기능 우선 순위 정하기.
4~10주차 -> 프로젝트 개발,   중간 6~7주차 -> 프로젝트 중간 점검
10~12주차 -> 프로젝트 개발 최종 점검 및 디버그 테스트,사용자 피드백 시간.
------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
다른 서비스와의 차이점 -> 1.책 표지를 촬영하면 책의 기록을 저장 가능하고 책의 간단한 정보를 얻을 수 있다.
				      2. 사용자가 기록한 책을 보고 AI로 사용자 맞춤형 추천 기능 제공.
				      3. AI가 독서 목표를 달성 가능하게 도와준다 -> EX):한달에 5권 이런느낌이 아니라, 난이도를 조절해가며 목표를 만들고 달성하게 도와준다.

------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

기대효과-> 1.사용자가 기록한 독서 데이터를 기반으로 개인 맞춤형 도서를 추천하여 더욱 효율적인 독서를 지원한다.
2.독서 챌린지 기능을 통해 사용자가 스스로 목표를 설정하고 꾸준한 독서 습관을 형성할 수 있다.
3.각 책의 핵심 줄거리 요약과 태그 기능을 제공하여 사용자가 책의 내용을 빠르게 이해하고 흥미로운 책을 쉽게 발견할 수 있도록 돕는다.
4.커뮤니티 를 이용해서 사용자 간의 여러 논의가 가능하다.

그리고 사용할 기술 중에서 책의 데이터를 가져오는건 google books api와 공공 도서 데이터에서 가져올것 같습니다.
------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

배경=>  사람들은 독서 습관을 기르는 것이 쉽지 않고, 책을 찾고 기록하는 과정도 번거롭다고 느낍니다. 기존 독서 앱은 책을 기록하는 과정이 불편하거나, 사용자의 독서를 효과적으로 도와주는 기능이 부족한 경우가 많습니다. 그래서 우리는 책 표지를 촬영하는 것만으로 간편하게 기록할 수 있고, AI를 이용해서 책의 정보를 제공하며, 사용자 맞춤형 도서를 추천하는 기능을 갖춘 독서 관리 앱을 만들고자 합니다. 또한, AI 기반 챌린지 기능을 통해 사용자가 꾸준히 독서를 이어갈 수 있도록 돕는 것이 목표입니다.
------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

기능  1. 책 표지를 촬영하면 책에 대한 정보와 사용자 페이지에 기록 하기
	2. 사용자의 기록을 이용해서 사용자 맞춤형 도서 추천
	3. AI기반 독서 챌린지 기능
	4. 유저간 커뮤니티 기능
	5. 사용자 독서 데이터의 통계 및 시각화
